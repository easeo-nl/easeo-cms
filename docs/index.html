JSON instead of MySQL

Secure by design

2FA, CSRF tokens, bcrypt, rate limiting, account lockout, CSP headers, SVG sanitization. Security is not a plugin.

Upload, visit the domain, complete the wizard. Company info, branding, content and admin account in five steps.

SEO

SEO out of the box

Auto-sitemap, JSON-LD structured data, meta tags, breadcrumbs, Open Graph. No plugin required.

i18n

Multilingual

Built-in i18n system with JSON language files. English and Dutch included. Add any language by creating a single file.

WordPress vs. EASEO CMS

WordPress EASEO CMS
Database required MySQL / MariaDB None — JSON files
Dependencies PHP + MySQL + plugins PHP only
Installation Create DB, wp-config, wizard Upload + wizard
Updates Core + theme + 10-30 plugins git pull
Security Depends on plugins Built-in: 2FA, CSRF, CSP
Load time DB queries + plugin overhead Direct file reads
Data sovereignty Often requires cloud/SaaS plugins 100% on your server
Ecosystem 60,000+ plugins Manual extension
Complex sites E-commerce, membership, LMS SMB websites, blogs

Architecture

index.php              Homepage
blog.php / blog-post.php   Blog engine
contact.php            Contact page
pagina-router.php      Dynamic pages

includes/              Core PHP (content, auth, forms, media, mail, i18n)
beheer/                Admin panel (dark theme)
templates/             Reusable frontend sections
lang/                  Language files (en.json, nl.json, ...)
data/                  JSON data files (git-ignored)
